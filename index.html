Loops, Conditional Statements, Functions, Variables, Parameters, Arrays, Associative Arrays

Javascript loops allow you to repeat a task multiple times. For example, you can print the numbers 1 through 100 without having to hard code them in.

    function countUp() {
        var d = document.getElementById("1to100");
        d.innerHTML = "";
        for(var i = 1; i <= 100; i++){
            d.innerHTML += i + ", ";
        }
    }

Conditional statements allow you run tasks based on if they meet certain conditions. For example, I can make every even number italicized.

    function countUp2() {
        var d = document.getElementById("1to100even");
        d.innerHTML = "";
        for(var i = 1; i <= 100; i++){
            if( i % 2 != 0){
                d.innerHTML += i + ", ";
            } else {
                d.innerHTML += "<em>" + i + "</em>, ";
            }
        }
    }

Functions allow you to seperate tasks into modularized chunks. My previous chunks of code demonstrate the syntax of a javascript function.

Variables function similar to variables in algebra. A value or resource is assigned to a variable, which can then be manipulated like the following:

    // Initializing the variable
    var a = 3;

    // Add to the variable;
    var a += 2; // a is now '5'

    // Set a whole new value
    var a = 10; // a is now '10'

    // Add variables, just like in algebra
    var b = 11;
    var c = a + b; // c is now 21

Parameters are the names in the () that allow arguments to be passed into a function, making the function useful for accomplishing a task with a give input.

    function add(parameter1, parameter2) {
        return parameter1 + parameter2;
    }

    var result = add(5,4); // result is now 9

Arrays allow you to store multiple items in one variable.

    var classes = ["CIT 261", "CS 235", "CS 308"];
    var firstClass = classes[0]; // firstClass is now 'CIT 261'

Associative Arrays are arrays with string indexes, like 'name' instead of a number like 4. JavaScript does not support associative arrays.

    var classes = [];
    classes["firstClass"] = "CIT 261";
    var x = classes.length; // Will return 0 
                            // because nothing 
                            // was set

Object Creation Functions, Inheritance, Properties, Methods, Instantiation

An object is a collection of properties, and a property is an association between a name (or key) and a value. (Source) An object can be very useful when dealing with complex amounts of information.

An object can be instantiated, or generated, by the following method:

    var Person = new Object();

Objects then can be given properties by the following method:

    Person.race = 'Hispanic';
    Person.name = 'Jared';
    Person.desireForaGoodGradeInCIT261 = 999;

Objects can be given methods as follows:

    var Person = {
        greet() {
            console.log("Hello, I am " + this.name + "!");
        }
    };

JSON Parse, Stringify

JavaScript Object Notation or JSON is a common way to exchange data on the internet between servers. A plain text JSON could arrive in the following way:

    '{ 
        "race":"Hispanic", 
        "name":"Jared", 
        "desireForaGoodGradeInCIT261": 999
    }'

Using JSON.parse(), we can convert the string into an object for easy data handeling:

   /* Note the JSON is on one line.
    * Breaking up the text in the text editor
    * will break the JSON string
    */
    var Person = JSON.parse('{"race":"Hispanic","name":"Jared","desireForaGoodGradeInCIT261": 999}');
    var d = document.getElementById("jsonParse");
    d.innerHTML = Person.name + " says Hi!";

Using JSON.stringify(), we can convert the object back into a string for easy data transfer:

    var Person = JSON.parse('{"race":"Hispanic","name":"Jared","desireForaGoodGradeInCIT261": 999}');
    var d = document.getElementById("jsonString");
    d.innerHTML = Person.name + " says Hi!<br>";
    var jsonString = JSON.stringify(Person);
    d.innerHTML += jsonString;

Using XMLHTTPRequest to Consume a JSON Web Service

XMLHTTPRequest sends a request to web server, which returns data. Useful for REST APIs and asynchronous loading

    function requestTest() {
        console.log("Request Test Running...");
        var xhttp = new XMLHttpRequest();
        xhttp.onreadystatechange = function() {
            if (this.readyState == 4 && this.status == 200) {
                var test = JSON.parse(xhttp.responseText);
                console.log(test);
                document.getElementById("requestTestResult").innerHTML = test.body;
            } else {
                document.getElementById("requestTestResult").innerHTML = "Error in request";
            }
        };
        xhttp.open("GET", 'https://jsonplaceholder.typicode.com/posts/1', true);
        xhttp.send();
    }

Local Storage API, Storing and Retrieving Simple Data, Arrays, Associative Arrays, and Objects

Local Storage is a way to store data locally on the client's machine. It provides a greater size limit than cookies. Because local storage only supports strings, arrays, associative arrays, and objects need to be serialized before storage, and parsed on retrival. 

    function localTest() {
        console.log("Local Storage Test Running...");
        // Storing in Local Storage
        localStorage.simpleData = "Hello world! ";
        localStorage.setItem("array", JSON.stringify(["blue", "orange", "red"]));
        var aa = {
            "love" : "chocolate",
            "like" : "to dance",
            "hate" : "loosing"
        };
        localStorage.setItem("associative", JSON.stringify(aa));
        var Person = new Object();
        Person.name = "Jared";
        Person.age = 23;
        localStorage.setItem("obj", JSON.stringify(Person));

        // Retriving from Local Storage
        var output = document.getElementById("localOutput");
        var savedPerson = JSON.parse(localStorage.getItem("obj"));
        output.innerHTML = localStorage.simpleData
            + "My name is " + savedPerson.name + ". "
            + "I am " + savedPerson.age + " years old. "
            + "My favorite colors are:<ul>";
        var savedArray = JSON.parse(localStorage.getItem("array"));
        savedArray.forEach(function(color) {
            output.innerHTML += "<li>"+color+"</li>";
        }, this);
        output.innerHTML += "</ul><br>";
        var savedAA = JSON.parse(localStorage.getItem("associative"));
        console.log(savedAA);
        Object.keys(savedAA).forEach(function(key) {
            output.innerHTML += "I " + key +" "+ savedAA[key] + ". ";
        }, this);
    }

Standard JavaScript Events Including those for Mobile Devices ( Ex. onTouchBegin, onLoad, etc.) and Animation and Transition Events

Javascript Events provides a way for javascript to react to different events such as a click or touch, making thinks like UI easier to program

    function eventTest() {
        var output = document.getElementById('eventTestOutput');
        var output2 = document.getElementById('eventTestOutput2');
        output.innerHTML = "<br><button class='btn' id='eventButton'>Click Me!</button><br>";
        output2.innerHTML = "<br><button class='btn' id='eventButton2'>Touch Me!</button><br>";
        var btn = document.getElementById('eventButton');
        var btn2 = document.getElementById('eventButton2');
        btn.onclick = function() {
            var rndCol = 'rgb(0,0,255)';
            output.style.backgroundColor = rndCol;
            output.innerHTML += "Clicked!<br>";
        }
        output2.onmouseover = function() {
            var rndCol = 'rgb(255,255,255)';
            output2.style.backgroundColor = rndCol;
            output2.innerHTML += "Mouse over!<br>";
        }
        output2.onmouseout = function() {
            var rndCol = 'rgb(0,255,0)';
            output2.style.backgroundColor = rndCol;
            output2.innerHTML += "Mouse out!<br>";
        }
        btn2.ontouchstart = function () {
            var rndCol = 'rgb(255,255,0)';
            output2.style.backgroundColor = rndCol;
            output2.innerHTML += "Touch Started!<br>";
        }
        btn2.ontouchend = function () {
            var rndCol = 'rgb(0,255,255)';
            output2.style.backgroundColor = rndCol;
            output2.innerHTML += "Touch Ended!<br>";
        }
        var move = document.getElementById("eventTestOutput3");

        move.style.animation = "move 4s 2";

        move.animationstart = function() {
            move.innerHTML = "animationstart event occured - The animation has started";
            move.style.background = "pink";
        }
        move.animationiteration = function() {
            move.style.background = "lightblue";
        }
        move.animationend = function() {
            move.style.background = "lightgray";
        }
    }

Designing, Defining, and Triggering CSS3 Transitions without Custom Libraries

CSS3 transitions allows you to change property values smoothly over time. This can make things like animating menu buttons and navigation much easier.

This css shows the difference between implementing a css change without vs. with transitions. Note how the css selectors such as :active and :hover are used to declare when the changes are to be made. The -webkit-tansition is used with transition to be compatible with most modern browsers. These prperties take the following parameters (from w3schools.com):

  • transition-property: Specifies the name of the CSS property the transition effect is for
  • transition-duration: Specifies how many seconds or milliseconds the transition effect takes to complete
  • transition-timing-function: Specifies the speed curve of the transition effect
  • transition-delay: Defines when the transition effect will start 
    /* No transition */
    #box0 {
        width: 100px;
        height: 100px;
        background: red;
    }

    #box0:hover {
        width: 300px;
    }

    #box0:active {
        background: blue;
    }

    /* Transition */
    #box1 {
        width: 100px;
        height: 100px;
        background: red;
        -webkit-transition: all 2s; /* For Safari 3.1 to 6.0 */
        transition: all 2s;
    }

    #box1:hover {
        width: 300px;
    }

    #box1:active {
        background: blue;
    }

and the html to call the css:

    <p>No transition</p>
    <div id="box0"></div>
    <p>Transition</p>
    <div id="box1"></div>

Check boxes can use transitions with the :checked selector. The syntax is a little different then expected for real time updating.

    #check0:checked + #check0 {
        font-weight: bold;
        font-size: 300%;
        color: red;
    }
    #check1:checked + #check1 {
        -webkit-transition: color 2s; /* For Safari 3.1 to 6.0 */
        transition: color 2s;
        font-weight: bold;
        font-size: 300%;
        color: red;
    }

Note, this time we are only animating the color and not all the properties like in the first example. Here is how we call the css in the html:

    <input type="checkbox" id="check0" class="check-with-label" />
    <label class="check-with-label" id="check0">No transition</label>
    <br>
    <input type="checkbox" id="check1" class="check-with-label" />
    <label for="check" class="check-with-label" id="check1">Only animate color</label>

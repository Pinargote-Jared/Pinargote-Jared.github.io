Loops, Conditional Statements, Functions, Variables, Parameters, Arrays, Associative Arrays

Javascript loops allow you to repeat a task multiple times. For example, you can print the numbers 1 through 100 without having to hard code them in.

    function countUp() {
        var d = document.getElementById("1to100");
        d.innerHTML = "";
        for(var i = 1; i <= 100; i++){
            d.innerHTML += i + ", ";
        }
    }

Conditional statements allow you run tasks based on if they meet certain conditions. For example, I can make every even number italicized.

    function countUp2() {
        var d = document.getElementById("1to100even");
        d.innerHTML = "";
        for(var i = 1; i <= 100; i++){
            if( i % 2 != 0){
                d.innerHTML += i + ", ";
            } else {
                d.innerHTML += "<em>" + i + "</em>, ";
            }
        }
    }

Functions allow you to seperate tasks into modularized chunks. My previous chunks of code demonstrate the syntax of a javascript function.

Variables function similar to variables in algebra. A value or resource is assigned to a variable, which can then be manipulated like the following:

    // Initializing the variable
    var a = 3;

    // Add to the variable;
    var a += 2; // a is now '5'

    // Set a whole new value
    var a = 10; // a is now '10'

    // Add variables, just like in algebra
    var b = 11;
    var c = a + b; // c is now 21

Parameters are the names in the () that allow arguments to be passed into a function, making the function useful for accomplishing a task with a give input.

    function add(parameter1, parameter2) {
        return parameter1 + parameter2;
    }

    var result = add(5,4); // result is now 9

Arrays allow you to store multiple items in one variable.

    var classes = ["CIT 261", "CS 235", "CS 308"];
    var firstClass = classes[0]; // firstClass is now 'CIT 261'

Associative Arrays are arrays with string indexes, like 'name' instead of a number like 4. JavaScript does not support associative arrays.

    var classes = [];
    classes["firstClass"] = "CIT 261";
    var x = classes.length; // Will return 0

Object Creation Functions, Inheritance, Properties, Methods, Instantiation

An object is a collection of properties, and a property is an association between a name (or key) and a value. (Source) An object can be very useful when dealing with complex amounts of information.

An object can be instantiated, or generated, by the following method:

    var Person = new Object();

Objects then can be given properties by the following method:

    Person.race = 'Hispanic';
    Person.name = 'Jared';
    Person.desireForaGoodGradeInCIT261 = 999;

Objects can be given methods as follows:

    Person.greet = function() {
        var d = document.getElementById("object");
        d.innerHTML = "Hello, my name is " + this.name +
            ". I am " + this.race + ". I have a " +
            this.desireForaGoodGradeInCIT261.toString() + 
            " desire to do well in this class.";
    }
    Person.greet();

Objects can also be defined in the following way:

    function Person(first, last, gender) {
        this.name = {
            first,
            last
        };
        this.gender = gender;
    };
    Person.prototype.greeting = function() {
        var d = document.getElementById("object2");
        d.innerHTML += "Hi!<br>";
    };
    Person.prototype.introduce = function() {
        var d = document.getElementById("object2");
        d.innerHTML += "I'm " + this.name.first + " "
                      + this.name.last + ".<br>";
    };

This way allows us to inherit objects like so:

    function Professor(first, last, gender, course) {
        Person.call(this, first, last, gender);
        this.course = course;
    }
    Professor.prototype = Object.create(Person.prototype);
    Professor.prototype.constructor = Professor;
    Professor.prototype.introduce = function() {
        var d = document.getElementById("object2");
        var title = "";
        switch (this.gender) {
            case "M":
                title = "Bro.";
                break;
            case "F":
                title = "Sis.";
                break;
            default:
                break;
        }
        d.innerHTML += "I'm " + title + " "
                      + this.name.last + ".";
    }

We can call the objects in the following way:

    function objectTest2(){
        // Base Class
        var me = new Person("Jared", "Pinargote", "M");
        // Inherited Class
        var prof = new Professor("Johan", "Baer", "M");
        // Base Methods
        me.greeting();
        me.introduce();
        // Inherited Method
        prof.greeting();
        // Modified Inherited Method
        prof.introduce();
    }

JSON Parse, Stringify

JavaScript Object Notation or JSON is a common way to exchange data on the internet between servers. A plain text JSON could arrive in the following way:

    '{ 
        "race":"Hispanic", 
        "name":"Jared", 
        "desireForaGoodGradeInCIT261": 999
    }'

Using JSON.parse(), we can convert the string into an object for easy data handeling:

   /* Note the JSON is on one line.
    * Breaking up the text in the text editor
    * will break the JSON string
    */
    var Person = JSON.parse('{"race":"Hispanic","name":"Jared","desireForaGoodGradeInCIT261": 999}');
    var d = document.getElementById("jsonParse");
    d.innerHTML = Person.name + " says Hi!";

Using JSON.stringify(), we can convert the object back into a string for easy data transfer:

    var Person = JSON.parse('{"race":"Hispanic","name":"Jared","desireForaGoodGradeInCIT261": 999}');
    var d = document.getElementById("jsonString");
    d.innerHTML = Person.name + " says Hi!<br>";
    var jsonString = JSON.stringify(Person);
    d.innerHTML += jsonString;

Using XMLHTTPRequest to Consume a JSON Web Service

XMLHTTPRequest sends a request to web server, which returns data. Useful for REST APIs and asynchronous loading

    function requestTest() {
        console.log("Request Test Running...");
        var xhttp = new XMLHttpRequest();
        xhttp.onreadystatechange = function() {
            if (this.readyState == 4 && this.status == 200) {
                var test = JSON.parse(xhttp.responseText);
                console.log(test);
                document.getElementById("requestTestResult").innerHTML = test.body;
            } else {
                document.getElementById("requestTestResult").innerHTML = "Loading";
            }
        };
        xhttp.open("GET", 'https://jsonplaceholder.typicode.com/posts/1', true);
        xhttp.send();
    }

Local Storage API, Storing and Retrieving Simple Data, Arrays, Associative Arrays, and Objects

Local Storage is a way to store data locally on the client's machine. It provides a greater size limit than cookies. Because local storage only supports strings, arrays, associative arrays, and objects need to be serialized before storage, and parsed on retrival. 

    function localTest() {
        console.log("Local Storage Test Running...");
        // Storing in Local Storage
        localStorage.simpleData = "Hello world! ";
        localStorage.setItem("array", JSON.stringify(["blue", "orange", "red"]));
        var aa = {
            "love" : "chocolate",
            "like" : "to dance",
            "hate" : "loosing"
        };
        localStorage.setItem("associative", JSON.stringify(aa));
        var Person = new Object();
        Person.name = "Jared";
        Person.age = 23;
        localStorage.setItem("obj", JSON.stringify(Person));

        // Retriving from Local Storage
        var output = document.getElementById("localOutput");
        var savedPerson = JSON.parse(localStorage.getItem("obj"));
        output.innerHTML = localStorage.simpleData
            + "My name is " + savedPerson.name + ". "
            + "I am " + savedPerson.age + " years old. "
            + "My favorite colors are:<ul>";
        var savedArray = JSON.parse(localStorage.getItem("array"));
        savedArray.forEach(function(color) {
            output.innerHTML += "<li>"+color+"</li>";
        }, this);
        output.innerHTML += "</ul><br>";
        var savedAA = JSON.parse(localStorage.getItem("associative"));
        console.log(savedAA);
        Object.keys(savedAA).forEach(function(key) {
            output.innerHTML += "I " + key +" "+ savedAA[key] + ". ";
        }, this);
    }

    // JS
    // Variables for referencing the canvas and 2dcanvas context
    var canvas,ctx;

    // Variables to keep track of the mouse position and left-button status 
    var mouseX,mouseY,mouseDown=0;

    // Variables to keep track of the touch position
    var touchX,touchY;

    // Draws a dot at a specific position on the supplied canvas name
    // Parameters are: A canvas context, the x position, the y position, the size of the dot
    function drawDot(ctx,x,y,size) {
        // Let's use black by setting RGB values to 0, and 255 alpha (completely opaque)
        r=0; g=0; b=0; a=255;

        // Select a fill style
        ctx.fillStyle = "rgba("+r+","+g+","+b+","+(a/255)+")";

        // Draw a filled circle
        ctx.beginPath();
        ctx.arc(x, y, size, 0, Math.PI*2, true); 
        ctx.closePath();
        ctx.fill();
    } 

    // Clear the canvas context using the canvas width and height
    function clearCanvas(canvas,ctx) {
        ctx.clearRect(0, 0, canvas.width, canvas.height);
    }

    // Keep track of the mouse button being pressed and draw a dot at current location
    function sketchpad_mouseDown() {
        mouseDown=1;
        drawDot(ctx,mouseX,mouseY,12);
    }

    // Keep track of the mouse button being released
    function sketchpad_mouseUp() {
        mouseDown=0;
    }

    // Keep track of the mouse position and draw a dot if mouse button is currently pressed
    function sketchpad_mouseMove(e) { 
        // Update the mouse co-ordinates when moved
        getMousePos(e);

        // Draw a dot if the mouse button is currently being pressed
        if (mouseDown==1) {
            drawDot(ctx,mouseX,mouseY,12);
        }
    }
    // Get the current mouse position relative to the top-left of the canvas
    function getMousePos(e) {
        if (!e)
            var e = event;

        if (e.offsetX) {
            mouseX = e.offsetX;
            mouseY = e.offsetY;
        }
        else if (e.layerX) {
            mouseX = e.layerX;
            mouseY = e.layerY;
        }
    }
    // Draw something when a touch start is detected
    function sketchpad_touchStart() {
        // Update the touch co-ordinates
        getTouchPos();

        drawDot(ctx,touchX,touchY,12);

        // Prevents an additional mousedown event being triggered
        event.preventDefault();
    }

    // Draw something and prevent the default scrolling when touch movement is detected
    function sketchpad_touchMove(e) { 
        // Update the touch co-ordinates
        getTouchPos(e);

        // During a touchmove event, unlike a mousemove event, we don't need to check if the touch is engaged, since there will always be contact with the screen by definition.
        drawDot(ctx,touchX,touchY,12); 

        // Prevent a scrolling action as a result of this touchmove triggering.
        event.preventDefault();
    }

    // Get the touch position relative to the top-left of the canvas
    // When we get the raw values of pageX and pageY below, they take into account the scrolling on the page
    // but not the position relative to our target div. We'll adjust them using "target.offsetLeft" and
    // "target.offsetTop" to get the correct values in relation to the top left of the canvas.
    function getTouchPos(e) {
        if (!e)
            var e = event;
        if(e.touches) {
            if (e.touches.length == 1) { // Only deal with one finger
                var touch = e.touches[0]; // Get the information for finger #1
                touchX=touch.pageX-touch.target.offsetLeft;
                touchY=touch.pageY-touch.target.offsetTop;
            }
        }
    }

    function resizeCanvas() {
        if(window.innerWidth < window.innerHeight){
            canvas.width = window.innerWidth * .7;
            canvas.height = canvas.width;
        } else {
            canvas.height = window.innerHeight * .7;
            canvas.width = canvas.height;
        }
    }
    // Set-up the canvas and add our event handlers after the page has loaded
    function init0() {
        // Get the specific canvas element from the HTML document
        canvas = document.getElementById('sketchpad');

        // If the browser supports the canvas tag, get the 2d drawing context for this canvas
        if (canvas.getContext)
            ctx = canvas.getContext('2d');

        // Check that we have a valid context to draw on/with before adding event handlers
        if (ctx) {
            // React to mouse events on the canvas, and mouseup on the entire document
            canvas.addEventListener('mousedown', sketchpad_mouseDown, false);
            canvas.addEventListener('mousemove', sketchpad_mouseMove, false);
            window.addEventListener('mouseup', sketchpad_mouseUp, false);

            // React to touch events on the canvas
            canvas.addEventListener('touchstart', sketchpad_touchStart, false);
            canvas.addEventListener('touchmove', sketchpad_touchMove, false);

            // Resize handler
            window.addEventListener('resize', resizeCanvas, false);
        }
        resizeCanvas();
    }

Designing, Defining, and Triggering CSS3 Transitions without Custom Libraries

CSS3 transitions allows you to change property values smoothly over time. This can make things like animating menu buttons and navigation much easier.

This css shows the difference between implementing a css change without vs. with transitions. Note how the css selectors such as :active and :hover are used to declare when the changes are to be made. The -webkit-tansition is used with transition to be compatible with most modern browsers. These prperties take the following parameters (from w3schools.com):

  • transition-property: Specifies the name of the CSS property the transition effect is for
  • transition-duration: Specifies how many seconds or milliseconds the transition effect takes to complete
  • transition-timing-function: Specifies the speed curve of the transition effect
  • transition-delay: Defines when the transition effect will start 
        /* No transition */
        #box0 {
            width: 100px;
            height: 100px;
            background: red;
        }

        #box0:hover {
            width: 300px;
        }

        #box0:active {
            background: blue;
        }

        /* Transition */
        #box1 {
            width: 100px;
            height: 100px;
            background: red;
            -webkit-transition: all 2s; /* For Safari 3.1 to 6.0 */
            transition: all 2s;
        }

        #box1:hover {
            width: 300px;
        }

        #box1:active {
            background: blue;
        }

and the html to call the css:

        <p>No transition</p>
        <div id="box0"></div>
        <p>Transition</p>
        <div id="box1"></div>

Check boxes can use transitions with the :checked selector. The syntax is a little different then expected for real time updating.

        #check0:checked + #check0 {
            font-weight: bold;
            font-size: 300%;
            color: red;
        }
        #check1:checked + #check1 {
            -webkit-transition: color 2s; /* For Safari 3.1 to 6.0 */
            transition: color 2s;
            font-weight: bold;
            font-size: 300%;
            color: red;
        }
